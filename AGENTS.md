# TaleLabs Agent Instructions

Before planning, editing, or implementing product work in this repository, read:

```txt
docs/talelabs-product-vision.md
```

That document defines what TaleLabs is, the MVP loop, the meaning of Generate, Assets, Projects, Brands, Products, Characters, and the later concepts that should not be pulled into the first billable loop without explicit direction.

Keep implementation aligned with the minimal MVP loop unless the user explicitly asks to expand scope:

```txt
Generate
Assets
Projects
Brands
Products
Characters
```

Credits and billing are core infrastructure, but they should live in account/header/billing UI rather than the main creative sidebar.
